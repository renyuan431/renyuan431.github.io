CCNA笔记——1、学习方法

发表于 2019-11-16 分类于 CCNA ， 路由交换方向 阅读次数：

一、总原则

4个W<What\Why\When\Where>和1个H

（一）What：这个技术是什么，搞清楚这项技术是什么。

（二）Why\When\Where：为什么什么时候在哪使用这项技术，搞清楚使用这项技术解决了现网中的哪些问题。

（三）How：如何使用这项技术，搞清楚这项技术的原理、结构和实现方法、流程。 多看书，多做实验是提升技术水平的真正方式，多敲命令是为了更好的理解技术，从根本上理解这项技术才是关键。关键字段的含义最好记住，即使无法一下理解也没关系，等以后学会抓包分析的时候就会慢慢理解了。

二、遇到问题如何找寻答案

如果周围有懂技术的朋友，请教他们是最快捷最直接的解决办法，但还是更建议通过搜索引擎、官方文档或做实验寻找答案，因为人的大脑可能记错答案但官方文档一般来说是不会出问题的，国内技术类网站都是中文，可以解决常见的一些疑问，但如果有比较冷门一些的问题，可能国内的中文网站已经很难找到解答，这时候思科的官方文档和官方社区是最好的求助对象，尤其是思科的官方论坛，经常有思科员工和技术大佬过去回答问题，是提升自我非常好的学习渠道，老外对技术细节的讨论十分到位，基本可以解决大部分疑惑。 搜索引擎尽量别用百度，搜索的效率相对低一些，容易浪费时间。

三、抓包有什么用？

有一些故障的现象并不明显，并不是看配置就能那么轻易的找到问题的答案的，需要去抓包分析，分析包的行为能帮助你给出一个排错的大概方向。一般来说 TCP用的比较多，IP头看分片，TOS字段比较多。TCP的话主要看TCP的一些机制和算法。

四、如何下载学习资料

推荐两个论坛，鸿鹄论坛以及攻城狮论坛。鸿鹄论坛上的考试题库更新比较快，包含版本比较新，也有一些学习资料；攻城狮论坛上的学习资料比较丰富，基本上所有需要的资料在攻城狮论坛都能下载到。

五、厂商的认证是否有必要考？

谈一下我浅薄的看法，对个人来说，应该将备考的过程视为提高自己技术水平的过程，在这个过程中加深对技术的理解和掌握，而不是指着证书带你走向人生巅峰，证书带不来高薪职位，技术水平才是王道。一般来说有个认证能够让企业的招聘者对你的技术水平有个大概的定位，在进入面试环节之前给你一点帮助，但如果进入面试以后面试官发现你名不副实，花大价钱考下来的证书就显得一点用没有了。
