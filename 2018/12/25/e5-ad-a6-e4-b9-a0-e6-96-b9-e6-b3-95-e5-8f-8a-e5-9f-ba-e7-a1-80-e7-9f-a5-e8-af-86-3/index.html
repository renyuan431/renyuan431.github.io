CCNA笔记——2、模拟器

发表于 2018-12-25 更新于 2019-11-16 分类于 CCNA ， 路由交换方向 阅读次数：

一、用什么模拟器

cisco packet tracer、GNS3或者EVE都可以。

（一）cisco packet tracer

cisco packet tracer 是思科出品的模拟器，目前是是7.2版（截至2018年12月18日），界面、功能和使用都非常简单，对新手十分友好，基本上一学就会。不足是这个模拟器相对简单，一些命令无法运行，不过基本能满足NA阶段的学习。

（二）GNS3

GNS3是一个于2008年发布的网络软件模拟器,目前是2.1.11版（截至2018年12月18日 ）。 它允许组合的虚拟和实际设备，用于模拟复杂的网络。 它使用Dynamips模拟软件来模拟Cisco IOS。 GNS3在很多大型公司中也有广泛的应用，例如埃克森公司、沃尔玛、AT&T和美国宇航局。功能上比cisco packet tracer 要丰富的多，但相对真机还是有一定差距且使用起来有一些比较麻烦的地方。

（三）EVE-NG

个人推荐使用EVE，因为EVE不仅可以模拟网络设备，也可以运行一切虚拟机，可以算得上是仿真虚拟环境，给用户最接近真实网络设备的体验；它是深度定制的Ubuntu操作系统，可以直接把它安装在x86架构的物理主机上。它也有ova版本，可以导入到VMware等虚拟机软件中运行。EVE-NG在交互模式上更加具有优势，与GNS3截然不同。GNS3更像是用户使用的软件，只有GNS3支持的 OS才能使用；而EVE-NG更像是CS模型，EVE-NG是服务端，用户端可以是支持http/https的任意OS。

二、EVE-NG安装及设置

（一）所需软件

EVE-NG的虚拟机文件、VMware-workstation（推荐用14以上版本，14之前的版本有安全隐患）、火狐浏览器、SecureCRT以及EVE-NG-Win-Client-Pack安装包。

EVE-NG的OVA文件链接：https://pan.baidu.com/s/1pVTvJt-1Pdm4MrHPQVcnQw 密码：nk1e

（二）安装步骤

1、安装VMware-workstation

一路下一步安装完成即可，序列号可以自己搜索，建议有条件的土豪同学尽量支持正版。安装完毕后进入程序主界面。

左侧菜单栏是目前已安装的虚拟机

2、导入虚拟机文件

VMware workstation安装完毕后，原来无法识别的虚拟机文件会出现VMware可识别的图标如下：

双击EVE-NG的虚拟机文件（一般是.OVA文件）后，弹出导入窗口，将虚拟机导入至本机。导入前务必确认硬盘有足够的剩余空间。

点击“编辑虚拟机设置”对虚拟机进行参数设置

拖动标尺对虚拟机可用内存进行配置，一般情况下建议最少分配4G内存给虚拟机，建议运行虚拟机的电脑内存最少8G，最好16G起步

设置CPU相关参数

在处理器一栏中，将虚拟化引擎中的第一项点选上，只要不是太老的设备都应支持此选项，有些机器需要在BIOS中打开相应选项；处理器数量根据自己CPU性能选择，如果不知道如何选择，搜索CPU-Z，下载后检测便知。

3、在BIOS中开启虚拟化功能

按F2或F12（进BIOS的按键有所不同，建议自行搜索本机进入BIOS方式）进入BIOS后，找到Configuration选项或者Security选项，然后找到Virtualization，或者Intel Virtual Technology选项，将此选项调整为enable后保存退出即可。

4、设置虚拟机参数

点击“开启此虚拟机”将虚拟机开机

虚拟机开机后，在火狐浏览器中，输入该地址即可登陆EVE模拟器

输入用户名：root，密码：eve后就能对系统进行设置

EVE-NG-Win-Client-Pack安装比较简单，一直下一步即可，会安装putty（连接设备的登陆工具，免费）以及抓包工具wireshark。步骤较为简单，在此略过。

5、改变EVE-NG通过DHCP分配的IP登陆地址

用于EVE模拟器无法找到IP地址或希望使用固定IP地址情况下使用，本人并未进行验证，完全是搬运老外的设置步骤，请谨慎使用。

输入命令：ifconfig pnet0 ，就可以看到目前分配的IP地址

rm -f /opt/ovf/.configured 将目前的IP地址删除，然后重启虚拟机，然后用admin\eve 作为用户名密码登陆EVE模拟器，随后进入设置模式

输入密码后，将DHCP设置为 static（静态）模式

因为在设置中，网卡采用的是NAT模式，所以分配的地址为virtual network editor下面设置的网段

输入想要的IP地址

输入子网掩码

网关输入不是一台现网中设备的地址，DNS输入同样地址

重启后，IP地址就是之前设置的IP地址

版本早一些的设置视频，英文讲解设置步骤

新版本设置视频，博主用的印度口音英文讲解

三、EVE-NG使用简略说明

（一）登陆

在火狐浏览器中输入模拟器启动后显示的IP地址 ，即可登陆EVE模拟器，用户名默认为admin，密码默认为eve

左边为菜单栏，此版本可以进行语言切换

（二）新建实验拓扑

点击菜单栏上的“Add an object”或在空白点击右键然后点击“Node”均可添加设备

一些EVE中不区分新建路由器和交换机，两种设备都在“cisco IOL”中，这时系统版本开头为L3的就是路由器，开头为L2的就是交换机，所有交换机均为三层交换机。

IOL（IOS on Linux）是Linux的可執行程式，其功能在於模擬Cisco IOS的執行。 對初學者而言IOL可模擬出的功能已非常足夠，IOL執行時所需的資源相對而言也非常低，且IOL的執行速度也非常快，建議選擇IOL。

若需IOL無法模擬出的功能時，可選擇可在Qemu模擬環境執行的qcow2鏡像檔，如Cisco vIOS L3及Cisco vIOS L2等。

可根据需要增加Ethernet以及Serial接口数量

添加路由器

添加交换机，所有添加的交换机均为三层交换机

添加虚拟PC

当鼠标放到设备的黄色插头图标时，插头图标会变大，此时点击设备并拖动到其他设备上即可完成设备线缆；连接

连线视频，下拉可选择连接不同的端口，由于录制视频的原因没录上

删除连线

开启所有设备，也可以单独选择一台设备，点击右键——开机。开机后右上角会有文字提示，设备图标也会点亮

_关闭所有设备，也可以单独选择一台设备，点击右键——关机。关机后右上角会有文字提示，设备图标会恢复暗色 _

EVE的开机过程和真机非常接近，需要输入no命令后进入命令行配置

（三）设置及保存开机启动配置

EVE模拟器的设备在没有启动配置的情况下，开机速度较慢，不方便实验，将已进行的配置保存为开机选项既能加快设备启动速度，也能保存实验进度。

（四）命令行杂记

show run | section XXX

show run 是查看所有运行在内存里的配置，在show run 后接管道符“|”（在英文输入状态下，按shift + 回车上面的键，即可输入管道符）section后面接所需要看的命令抬头，比如可以接 ip route ，表示只查看静态路由的相关配置。

int range f0/0 - 2

range 命令可以同时进入多个接口，进行普遍性的操作配置
